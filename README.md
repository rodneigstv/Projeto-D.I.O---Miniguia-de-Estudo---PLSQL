# 🚀 PLSQL com Inteligência Artificial

## 📌 Contexto e Objetivos
Este projeto faz parte do desafio da DIO e tem como propósito explorar a integração entre **Oracle PLSQL** e **Inteligência Artificial**.  
O objetivo é compreender como a IA pode otimizar o trabalho diário com PLSQL, oferecendo suporte em tarefas repetitivas, geração de código, troubleshooting e até mesmo na curadoria de conhecimento.  

Ao final, este repositório será um **caderno temático** que demonstra não apenas domínio técnico, mas também pensamento crítico e capacidade de estruturar conhecimento de forma clara e reutilizável.

## 🤖 Como a Inteligência Artificial facilita a escrita e otimização de queries em PLSQL

### 1. Escrita e Geração Automática
- **Text-to-SQL**: Ferramentas como o framework *Vana* e o *AI Assistant* permitem gerar consultas SQL completas a partir de pedidos em linguagem natural.  
- **Autocompletar e Sugestões**: Assistentes como o *GitHub Copilot* e o *Oracle Code Assist* funcionam como "programadores de par", sugerindo códigos e completando trechos parciais para reduzir erros manuais.  

### 2. Otimização de Performance
- **Melhores Práticas**: A IA recomenda o uso de índices, subqueries eficientes e a redução de chamadas desnecessárias para acelerar as consultas.  
- **Eficiência em PL/SQL**: Sugere práticas avançadas como o uso de `BULK COLLECT` e a redução de *context switches* para otimizar o uso de memória em cursores.  
- **Identificação de Gargalos**: Ferramentas analisam o código para detectar ineficiências, bugs e sugerir refatorações automáticas.  

### 3. Contexto e Precisão (RAG)
- **Conhecimento do Esquema**: Para evitar "alucinações", a IA utiliza metadados (definições de tabelas, chaves e restrições) para garantir que os *joins* e filtros estejam corretos.  
- **Técnica de RAG**: Através da *Geração Aumentada de Recuperação (RAG)* e bancos de dados vetoriais, a IA consulta o modelo de dados específico da empresa antes de gerar a resposta.  

### 4. Documentação e Manutenção
- **Explicação de Código**: A IA pode explicar a lógica de queries complexas e gerar comentários automáticos, melhorando a legibilidade e a manutenção do banco de dados.  
- **Documentação Automatizada**: Criação de resumos e guias de uso diretamente a partir do código, facilitando a transferência de conhecimento entre equipes.

- **PLSQL**: linguagem procedural da Oracle para manipulação de dados.  
- **NotebookLM**: ferramenta de organização e estudo com IA.  
- **Prompt**: instrução ou pergunta feita à IA para gerar respostas.  
- **Troubleshooting**: processo de identificar e resolver problemas técnicos.  
- **GitHub Copilot**: assistente de código baseado em IA.  
- **Autonomous Database**: banco de dados Oracle com automação inteligente.
